ВАЖНО
======
Это демонстрационная версия проекта для показа кода по вакансии.
Репозиторий существенно урезан: часть модулей и служебного кода удалена.
Также в комплекте отсутствует дамп базы данных.
Поэтому полноценный запуск и работа всех функций в текущем виде не гарантируются.

AntiSpam Shield — описание проекта
==================================

1) Что это
----------
AntiSpam Shield — система защиты форм и API от спама и ботов без классической капчи.
По идее и поведению похожа на reCAPTCHA v3: пользователь не проходит визуальные проверки,
а система оценивает вероятность "человек/бот" по поведенческим сигналам.

2) Для чего нужен проект
------------------------
- Снизить количество спам-заявок в формах (обратная связь, регистрация, заявки).
- Уменьшить нагрузку от автоматических бот-скриптов.
- Оставить UX чистым: без "выберите все светофоры" и прочих интерактивных капч.
- Дать администратору гибкое управление через правила фильтрации и аналитику.

3) Как это работает (кратко)
----------------------------
1. На сайт подключается JavaScript SDK (`/sdk.js?key=PUBLIC_KEY`).
2. SDK собирает поведенческие признаки (мышь, клавиатура, скролл, тач, fingerprint, headless).
3. SDK получает токен через API (`/api/v1/token`) и добавляет его в форму.
4. Бэкенд вашего сайта отправляет токен в AntiSpam API (`/api/v1/verify`) с PRIVATE_KEY.
5. API возвращает результат (`success`, `score`, признаки фильтрации).
6. На основе score и правил принимается решение: пропустить или отклонить запрос.

4) Почему именно такой подход
-----------------------------
- Невидимость для пользователя: меньше трения, выше конверсия форм.
- Вероятностная модель (score 0.0..1.0) лучше адаптируется к "серым" ботам.
- Правила фильтрации закрывают кейсы, где одного score недостаточно
  (IP/CIDR, User-Agent, заголовки, пороговые ограничения и т.д.).
- Кастомные поля и логи позволяют анализировать спам по источникам и сценариям.

5) Архитектура проекта
----------------------
- Backend: Go (единая точка входа, REST API, SDK-эндпоинт, reverse proxy).
- Admin панели:
  - Vue 3 SPA (`/vue/`) — через Go API и JWT.
  - PHP (`/php/`) — отдельный backend, сессии.
  - Python Flask (`/python/`) — отдельный backend, сессии.
- База данных: PostgreSQL / SQLite / MySQL (переключается конфигурацией).
- Docker: all-in-one контейнер (Go + PHP + Python + PostgreSQL).

6) Основные функции
-------------------
- Антиспам-скоринг 0.0..1.0.
- Headless-детекция (Selenium/Puppeteer и аналоги).
- Мультиусловные правила фильтрации (AND-логика).
- Кастомные поля проекта для сегментации логов/статистики.
- Логи верификаций с фильтрами и деталями.
- Дашборд статистики (периоды, фильтры, графики).
- Мультиязычность интерфейсов (RU/EN).

7) Быстрый запуск
-----------------
Вариант A (рекомендуется): Docker

  docker-compose up --build

После запуска:
- Главная: http://localhost:8080/
- Vue admin: http://localhost:8080/vue/
- PHP admin: http://localhost:8080/php/
- Python admin: http://localhost:8080/python/

Логин по умолчанию: admin / admin

Вариант B: локальный запуск
- Запустить Go сервер: go run ./cmd/server/
- При необходимости отдельно поднять PHP и Python админки
  (см. SETUP.md и README.md).

8) Интеграция на клиентский сайт
--------------------------------
1. Создать проект в админке и получить ключи (public/private).
2. Подключить SDK:
   <script src="http://your-host:8080/sdk.js?key=YOUR_PUBLIC_KEY"></script>
3. На сервере формы вызывать `/api/v1/verify` с `secret` (private key) и `token`.
4. Отклонять запросы с низким score или попавшие под блокирующие правила.

9) Где что лежит
----------------
- `cmd/server/` — запуск Go сервера.
- `internal/` — бизнес-логика Go (handlers/services/repository/scoring).
- `admin/vue/` — Vue 3 админка.
- `admin/php/` — PHP админка.
- `admin/python/` — Flask админка.
- `migrations/` — SQL миграции для SQLite/PostgreSQL/MySQL.
- `sdk/` — исходники JS SDK.
- `deploy/` — Dockerfile и служебные файлы контейнера.

- Команды, которым нужна self-hosted альтернатива внешним anti-bot сервисам.

Итог
----
Проект решает задачу "минимум трения для реальных пользователей + контроль ботов
через скоринг и правила". Подходит как для быстрого внедрения (Docker), так и
для гибкой адаптации под свою инфраструктуру.
